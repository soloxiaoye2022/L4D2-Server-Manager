#### 安装教程
1.  先把全部文件上传到服务器内的服务端文件（和js-mods一样放置）
2.  cd到服务端文件下（也就是脚本所在的目录）
3.  运行js-mods.sh脚本，命令：
```bash
./js-mods.sh
```
也可以不用cd到目录下，直接用命令：
```bash
*/js-mods.sh
```
其中的`*`为你脚本所在的目录路径

4.  按照提示，输入插件名所对应的数字即可

#### 注意事项
1.  脚本必须和win系统放置js-mods软件一样放置在服务端文件内，并且必须也要上传`JS-MODS`文件夹（自行去豆瓣酱整合包内提取）
2.  如果你用脚本安装过插件，则不能删除文件`plugins.txt`，如果出错，自行负责（但，你也可以修改`plugins.txt`文件内容，修改成JS-MODS内插件文件夹所对应的名字即可，一行只能写一个插件，并且要用linux的换行符换行）
3.  `JS-MODS`文件夹内的插件文件名不能包含空格，需要改成`_`或者`-`（如果你不想一个个改，可以运行脚本`rename.sh`）
4.  如果`JS-MODS`文件夹内的插件文件名中包含空格，则会出错（系统的原因），如果出错了自行负责

#### 补充
1.  此脚本已经合并于本人的开服脚本，如果想开服和安装插件不分开脚本使用，则自行去下载本人发布的开服脚本
2.  `rename.sh`脚本作用是自动修改JS-MODS文件夹内所有文件夹名字中含有空格的字符，修改为_
3.  `digit.sh`脚本是作用是自动排列数字并打印到控制台方便复制/修改，`start`是开始数字，`end`是结尾数字，两两数字之间用`;`隔开，方便安装时输入数字过多而导致浪费时间
